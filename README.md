# Бот учёта рабочего времени → Google Sheets (вебхук, без сервера)

Бот слушает тему "приход/уход" в форум-группе Telegram через **вебхук** —
без постоянно работающего сервера (VPS), полностью бесплатно на Vercel.

## Как это работает

1. Сотрудники весь день пишут в теме строго: **"Пришел HH:MM"** / **"Ушел HH:MM"**
   (формы слова любые: пришел/пришёл/пришла/пришли, ушел/ушёл/ушла/ушли).
   Каждое такое сообщение мгновенно прилетает на вебхук и копится в скрытом
   листе `_state` твоей Google-таблицы — без немедленной реакции бота.
2. В конце дня, когда всё посчитано (касса и т.п.) и админ сказал всем
   написать "Ушел" — админ пишет в этой же теме команду **`/итоги`**.
3. Бот мгновенно:
   - забирает всё накопленное за день,
   - сшивает пары "Пришел→Ушел" по каждому сотруднику,
   - считает часы,
   - подтягивает Имя/Ставку/Позицию из листа **"Персонал"** по Telegram-аккаунту,
   - записывает строки в лист **"Daily"**: Дата, Имя, Смена, Ставка, Аванс=0,
     Штраф=0, Касса=0, Итог=формула (по позиции — повар/бариста — Ставка×Смена;
     админ/официант — спецформулы с %% от кассы),
   - отвечает в чат, сколько смен записано,
   - если у кого-то нет пары (забыл написать "Ушел") — это остаётся в буфере
     до следующего `/итоги`.

Кассу/Аванс/Штраф вписывает админ вручную в таблице — бот их не трогает.

## Структура проекта

```
api/webhook.py       — точка входа, сюда Telegram присылает сообщения (Vercel)
core.py               — разбор сообщений и сшивка пар "Пришел/Ушел"
parser.py             — regex-парсер формата "Пришел/Ушел HH:MM"
sheets.py             — работа с Google Sheets (буфер, Daily, Персонал)
telegram_api.py        — отправка ответных сообщений в Telegram
config.py             — все настройки из переменных окружения
setup_webhook.py      — одноразовый скрипт регистрации вебхука
```

## Установка

### 1. Telegram-бот

- `TELEGRAM_BOT_TOKEN` — токен от @BotFather
- Отключи Privacy Mode (`/setprivacy` → бот → `Disable`)
- Бот должен быть **админом группы**
- `TARGET_CHAT_ID` / `TARGET_TOPIC_ID` — узнать командами `/chatid` и `/topicid`
  (можно временно вернуть старый bot.py для этого, либо посмотреть в ссылке
  на сообщение в Telegram Desktop/Web — там виден `thread_id`)

### 2. Google Sheets

1. В [Google Cloud Console](https://console.cloud.google.com/) включи **Google Sheets API**
2. Создай Service Account, скачай JSON-ключ
3. Открой свою таблицу → "Настройки доступа" → дай доступ **Редактор** на
   email сервис-аккаунта (из JSON, `...iam.gserviceaccount.com`)
4. Скопируй **ID таблицы** из URL (между `/d/` и `/edit`)
5. Убедись, что есть листы **"Daily"** (Дата|Имя|Смена|Ставка|Аванс|Штраф|Касса|Итог)
   и **"Персонал"** (Имя|Telegram|Позиция|Ставка)

### 3. Деплой на Vercel

1. Залей папку проекта на GitHub (новый репозиторий)
2. На [vercel.com](https://vercel.com) → New Project → выбери этот репозиторий → Deploy
3. В настройках проекта (Settings → Environment Variables) добавь:

```
TELEGRAM_BOT_TOKEN=...
TARGET_CHAT_ID=...
TARGET_TOPIC_ID=...
ADMIN_IDS=твой_telegram_user_id
TIMEZONE=Asia/Bishkek
GOOGLE_SHEET_ID=...
GOOGLE_SERVICE_ACCOUNT_JSON={"type":"service_account",...}   ← весь JSON-файл целиком, одной строкой
GOOGLE_SHEET_WORKSHEET=Daily
GOOGLE_PERSONNEL_WORKSHEET=Персонал
```

4. После деплоя Vercel даст ссылку вида `https://твой-проект.vercel.app`
5. **Зарегистрируй вебхук** (один раз, с любого компьютера с Python):
```bash
pip install requests python-dotenv
python setup_webhook.py https://твой-проект.vercel.app/api/webhook
```
Должен ответить `{"ok": true, "result": true, ...}`.

## Команды в чате

| Команда | Кто | Что делает |
|---|---|---|
| `Пришел HH:MM` / `Ушел HH:MM` | сотрудник | копится в буфере, без мгновенного ответа |
| `/итоги` | админ (из `ADMIN_IDS`) | обрабатывает весь накопленный буфер, пишет в Daily |

## Ограничения

- Время вокруг полуночи ("Ушел 23:50" в сообщении, отправленном в 00:10)
  автоматически сдвигается на предыдущий день.
- Если сотрудник написал "Пришел" дважды без "Ушел" между ними — первый
  потеряется (уйдёт в leftover без пары, не попадёт в Daily). Такое будет
  видно, если после `/итоги` бот скажет "Не закрыто: N" — стоит проверить
  лист `_state` и поправить вручную при необходимости.
- Лист `_state` — служебный, не трогай его руками без необходимости (там
  хранится буфер несшитых событий между вызовами `/итоги`).
